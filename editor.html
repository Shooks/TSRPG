A word on chance and events.

As you might have noticed chance does not mean what you might think it does when triggering multiple events. The reason is because there is a limitation on how many events can be ran, i.e. one. Instead what I have decided to do is to implement a system that has a chance of adding the event to an array(An array is a data structure that contains a group of elements). After the event ID's have been added to the array it is randomized and the first value in the array is picked as the "winning" event ID to trigger.

IDrequired

The ID is a unique indentifier of all items in data.xml. The parser will not load an item without an ID or with a duplicated ID. ID is always a number.

Namerequired

The name is a human indentifier, in other words to make it simpler for humans to read and understand what is going on.

Pricerequired

The amount that has to be paid to purchase an item. Sell will per default be half that of price. Price is always a number.

Threat

Defines the chance of triggering combat with a enemies that have been defined. If no enemies have been defined then threat make no difference. The value is a number between 0-100.

Base Healthrequired

Defines the multiplier of health that an enemy has. The value is always a number. The equation used to calculate the enemies health is:
Enemy base health + (Enemy base health / 8) * player level.

Base Damagerequired

Defines the multiplier of damage that an enemy has. The value is always a number. The equation used to calculate the enemies damage is:
Enemy base damage + (Enemy base damage / 10) * player level.

Startwith

Defines whether or not a player start with the location discovered. The value is either empty, 1 or true. 1 and true have the same effect.

Descriptionrequired

Provides a description to help the player understand what is going on.

On Travel Messagerequired

The message that is shown when a player travels to a location.

Master

Defines the ID of a locations master. When traveling to locations with the master value set, the option of returning to its master is presented.

Allowed Genders

Gender is used when creating an enemy. If you want to prohibit a specific enemy from becoming a certain gender then define the one or two genders that are allowed here.

Discoverables

Defines the ID of what locations can be discovered when visiting a location.

Events

Defines the ID of what events can be triggered.

Button

Defines what buttons can be showed. Post-parser formating is -> function;id;name.

Requirement

Defines what criteria must be met before an event can be triggered.

Effects

Defines what effects(e.g. getting healed, increased xp) will trigger when using a potion or triggering an event.

Children

Defines the IDs of locations that can be visited without having to be discovered from a certain location.

Sell

Defines what items are sold by a vendor.

Max Run

Define how many times an event can trigger.

Gender

The gender of a character.

onloss

Defines what event will be ran when the player lose combat. One event will randomly be selected and triggered.

onwin

Defines what event(s) will become visible when the player win combat.

onmaxlust

Defines what event will be ran when the player lose combat by having their lust maxed. One event will randomly be selected and triggered.

Hit/Crit chance

Defines, in a value between 0-100, what the odds are of an enemy hitting/Critting the player in combat. 0 meaning 0%, 100 meaning 100% chance.

Crit Multiplier

Defines how much the enemy damage should be multiplier when it crit. Default value is 1.5, i.e. 150% more damage. Be careful when changing this value.

Rarity

Value between 0-200. Decides what color an item will have.

Dots

Defines what attack(s) will be run on every enemy turn.

Item Level

Number describing the combined value of an item. Determines the sell value of the item.
